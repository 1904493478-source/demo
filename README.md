# Arkme Demo

这是一个移动端优先的前端 Demo，核心展示「安排」模块。

「安排」用于统一收纳所有还没有发生、但之后可能需要用户处理的事情。它不是传统意义上单一的待办、日程、提醒或项目管理，而是把这些未来事项放进同一个低压力的入口里：提醒用户，但不压迫用户。

## 当前实现

- 手动创建、编辑、删除安排。
- 支持三种时间形式：没有时间、截止时间、时间段。
- 支持提醒时间，但提醒不等于完成。
- 首页按「今天」「近期」「没有时间」「以后再说」「已完成」分组展示。
- 已完成安排默认只显示最近 3 条，可展开更多，避免页面无限变长。
- 日历总览默认展示当天安排，点击日期后切换下方当天列表。
- 逾期不标红、不制造恐慌；用户可以把事项放入「以后再说」。
- 长按安排打开低压主动作：今天/近期可放到以后再说，已完成可标记为还没完，以后再说可放回今天。
- 左滑安排露出删除动作，删除前二次确认。
- 第一次进入「安排」时显示一次性教程，说明这个模块的用途和主要操作。

## AI 能力

AI 识别是隐式的：聊天里识别到候选安排时，不在聊天窗弹出打扰式提示，而是用轻微反馈提示用户。

当前支持：

- 发给自己的消息识别，例如「后天去一趟医院」。
- 私聊上下文识别，例如对方请求后，用户回复「好的」「可以」「没问题」等，会生成候选安排。
- 群聊只展示和当前用户相关的安排候选。
- 多物品请求识别，并合并到一个安排中。
- 同主题、同场景、时间相近的候选归集为一条安排，详情保留原始对话上下文。
- 候选安排必须由用户确认后才会成为正式安排。
- 后续对话可触发智能完成建议，例如「资料已经拿回来了」会让对应安排的完成按钮轻微闪烁，但不会自动替用户完成。
- 详情中展示来源可信度、原始上下文、冲突时间提示。
- 支持 AI 执行能力分层：
  - 只能用户完成。
  - AI 可辅助。
  - AI 可自动执行。
- AI 辅助目前只生成草稿：准备材料、草拟回复、生成路线，不会自动发送消息、打开地图或执行外部操作。
- 自动执行只做产品边界展示，必须具备权限、执行前确认和可回滚记录后才能接入真实服务。

## 本地测试入口

```sh
pnpm install
pnpm dev
```

打开：

- 移动端 Demo：http://127.0.0.1:5173/
- 消息测试台：http://127.0.0.1:5173/sendtest

如果 5173 端口被占用，Vite 会自动切到下一个可用端口，请以终端输出为准。

## AI 配置

1. 打开移动端 Demo。
2. 进入「安排」。
3. 点击顶部的「AI」入口。
4. 填入 OpenAI-compatible API 信息：
   - Base URL。
   - Model。
   - API Key。
5. 保存后进行连接测试。

说明：

- 当前调用的是 OpenAI-compatible `/chat/completions` 接口。
- API Key 仅保存在浏览器 localStorage 中。
- Token 消耗来自用户自己配置的模型服务。
- AI 识别只生成候选或建议，关键状态变更仍需要用户确认。

## 推荐验证流程

1. 首次点击「安排」，确认出现一次性教程。
2. 点击「新建」，手动创建一条安排，验证列表和日历能展示。
3. 编辑这条安排，修改时间、地点、备注，再保存。
4. 长按今天或近期安排，验证可放入「以后再说」。
5. 在「以后再说」长按安排，验证可放回今天。
6. 左滑安排，验证删除动作出现，并且二次确认后才删除。
7. 创建或确认一条安排后，发送后续聊天「资料已经拿回来了」，验证对应安排右侧完成按钮轻微闪烁。
8. 点击闪烁的完成按钮，验证只有用户确认后才进入已完成。
9. 已完成超过 3 条后，验证「展开更多」可展示剩余已完成安排。
10. 点击日历不同日期，验证下方只显示所选日期的安排。
11. 打开消息测试台，验证私聊、群聊、自发消息能生成 AI 候选。
12. 打开安排详情，点击「准备材料」「草拟回复」「生成路线」，验证只生成草稿，不自动执行外部动作。

## 产品原则

这个模块的目标不是把人变成任务机器，而是在真实生活不断变化时，帮用户轻轻接住那些之后可能要处理的事情。

因此：

- 不用红色逾期恐吓。
- 不自动替用户完成。
- 不把 AI 识别结果直接变成正式安排。
- 不让提醒和完成混为一谈。
- 允许「以后再说」，并把它作为正常操作，而不是失败状态。
