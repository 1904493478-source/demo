# Arrangements V0.5 AI Execution Plan

## 阶段目标

V0.5 最后阶段从最小安全闭环开始：模型可辅助判断安排是否“可能已完成”，但不能自动替用户完成；同时在安排详情里展示执行能力分层，明确哪些事只能用户完成，哪些事 AI 可辅助，哪些事未来可自动执行。

## 阶段状态

Slice 1 已实现并通过验证。当前稳定边界是：AI 可在本地完成识别未命中时辅助判断“可能已完成”，但仍不自动完成；安排详情可展示执行能力分层；自动执行只展示安全要求，不连接外部服务。

Slice 2 已实现并通过目标验证：AI 可辅助只生成当前详情内的草稿结果，不自动发送、不打开地图、不修改安排状态。

## Slice 1 范围

- 本地完成识别未命中时，调用用户配置的 OpenAI-compatible API 做完成匹配。
- 模型命中后只把安排置为 `可能已完成`，`completedAt` 保持为空。
- 安排详情展示三类执行能力：
  - `只能用户完成`：需要用户实际执行或亲自确认。
  - `AI 可辅助`：可准备材料、草拟回复、生成路线。
  - `AI 可自动执行`：本轮只展示安全要求，不真正执行。
- 自动执行必须展示安全边界：需要授权、执行前确认、保留可回滚记录。

## Slice 2 范围

- 在 `AI 可辅助` 详情里，辅助项从静态标签升级为可点击动作。
- `准备材料` 生成办理/出门前检查清单。
- `草拟回复` 生成可复制的回复草稿。
- `生成路线` 生成路线规划建议。
- AI API 已配置时可调用模型生成辅助内容；未配置或失败时使用本地模板。
- 辅助结果只展示在当前详情中，不修改安排状态，不调用外部服务。

## 暂不包含

- 真正调用地图、日历、消息发送或第三方服务执行动作。
- 自动把安排改为已完成。
- 执行能力编辑器。
- 后台定时扫描所有聊天记录。
- 辅助结果长期保存或跨设备同步。

## 验证重点

- AI fallback 请求失败或未配置时，不影响本地完成识别和手动确认。
- AI fallback 命中后只进入 `可能已完成`。
- 详情页能区分三种执行层级，并展示辅助动作或自动执行安全要求。
- 点击 AI 辅助动作只生成草稿，不触发完成、延期、编辑或外部动作。
